    SSH-Audit

    SSH-Audit

    SSH-Audit és una eina útil per revisar i avaluar la seguretat de la configuració del servei SSH. A través d'ella, pots identificar vulnerabilitats o configuracions desaconsellades, permetent-te millorar la seguretat del teu servidor SSH.

    Activitat pràctica: Auditar la configuració SSH amb ssh-audit

    Objectiu:

    Utilitzar l'eina ssh-audit per analitzar la configuració d'OpenSSH d'un servidor, identificar vulnerabilitats potencials, protocols insegurs, o configuracions millorables, i aplicar els canvis suggerits per millorar la seguretat.

    Requisits previs:

    • Un servidor Linux amb SSH habilitat (Ubuntu, Debian, CentOS, etc.).
    • Accés com a superusuari (root) o permisos de sudo.
    • ssh-audit instal·lat al client des del qual farem la comprovació.

    Passos de l'activitat:

    1. Instal·lació de ssh-audit

    ssh-audit es pot instal·lar fàcilment en qualsevol sistema Linux. També es pot utilitzar sense instal·lació simplement executant-lo com a fitxer de Python.

    • Ubuntu/Debian:

      Bash
      1
      sudo apt-get install ssh-audit

    • Alternativa amb pip (qualsevol distribució): Si tens pip instal·lat, pots fer-ho amb:

      Bash
      1
      sudo pip install ssh-audit

    O pots descarregar el fitxer ssh-audit.py directament des del seu repositori oficial.

    2. Comprovació bàsica de l'SSH del servidor

    Un cop tinguis ssh-audit instal·lat, pots executar-lo des del teu ordinador client per auditar el servidor SSH remot.

    Executa la següent comanda des del terminal del client:

    Bash
    1
    ssh-audit ip_del_servidor
    • Exemple:
      Bash
      1
      ssh-audit 192.168.1.100

    Aquesta comanda analitzarà la configuració del servidor SSH remot i proporcionarà un informe detallat que inclourà:

    • Versions del protocol SSH suportades.
    • Algorismes de xifrat, autenticació i intercanvi de claus.
    • Aspectes obsolets o insegurs (com l'ús del protocol SSH-1).
    • Recomanacions per millorar la configuració de seguretat.

    3. Interpretar els resultats de l'auditoria

    Un cop finalitzada l'execució de ssh-audit, obtindràs una sortida que contindrà informació estructurada en diferents seccions. Aquí tens alguns punts claus a observar:

    • Protocol: Si es detecta SSH-1, hauries de deshabilitar-lo, ja que és considerat obsolet i insegur.

    • Algorismes d'intercanvi de claus: Et mostrarà si el servidor està utilitzant algorismes segurs com curve25519-sha256 o diffie-hellman-group-exchange-sha256. Si s'estan utilitzant algorismes més febles com diffie-hellman-group1-sha1, es recomana substituir-los.

    • Xifrats (Ciphers): Revisar si s'utilitzen ciphers segurs com aes128-ctr, aes256-ctr i evitar xifrats insegurs com 3des-cbc o arcfour.

    • Algorismes de hashing (MACs): Comprovar si el servidor utilitza algoritmes de hash segurs com hmac-sha2-256 o hmac-sha2-512 i evitar hmac-md5 o hmac-sha1.

    • Claus d'host: És important assegurar-se que s'utilitzen claus segures. El format ed25519 és molt recomanable.

    4. Aplicar els canvis al servidor segons els resultats

    Un cop interpretats els resultats de l'auditoria, pots aplicar les millores suggerides editant el fitxer de configuració SSH al servidor.

    • Edita el fitxer /etc/ssh/sshd_config:
      Bash
      1
      sudo nano /etc/ssh/sshd_config

    Fes els següents ajustos segons el que ssh-audit et recomani:

    • Deshabilitar el protocol SSH-1 (si s'està utilitzant):

      Bash
      1
      Protocol 2

    • Actualitzar els ciphers per utilitzar els algorismes segurs:

      Bash
      1
      Ciphers aes128-ctr,aes192-ctr,aes256-ctr

    • Actualitzar els MACs per assegurar-te que utilitzes algorismes segurs:

      Bash
      1
      MACs hmac-sha2-256,hmac-sha2-512

    • Deshabilitar algorismes de clau febles i afegir suport per claus més segures com ed25519:

      Bash
      1
      HostKey /etc/ssh/ssh_host_ed25519_key

    5. Reiniciar el servei SSH

    Després de fer els canvis al fitxer de configuració, reinicia el servei SSH perquè els nous ajustos tinguin efecte:

    Bash
    1
    sudo systemctl restart sshd

    6. Repetir l'auditoria per verificar les millores

    Després d'haver aplicat els canvis de seguretat al servidor SSH, pots executar una altra auditoria amb ssh-audit per assegurar-te que els ajustos són efectius i que les vulnerabilitats han estat corregides.

    Bash
    1
    ssh-audit ip_del_servidor

    Si els ajustos han estat correctament aplicats, hauràs eliminat els elements insegurs que apareixien a l'auditoria inicial.

    7. Prova de connexió

    Per acabar, fes una prova de connexió SSH des d'un client per verificar que la configuració no ha bloquejat l'accés correcte. Simplement utilitza la comanda SSH per accedir al servidor com ho fas normalment:

    Bash
    1
    ssh usuari@ip_del_servidor
