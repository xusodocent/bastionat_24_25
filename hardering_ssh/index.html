    Hardering SSH

    Hardering d'un servidor SSH

    El hardening de SSH és una sèrie de pràctiques per augmentar la seguretat de les connexions SSH al teu sistema. A continuació tens alguns passos importants per fer un hardening de SSH:

    1. Canviar el port per defecte

    Per defecte, SSH escolta al port 22. Canviar-lo pot reduir els intents d'atacs automàtics.

    • Edita el fitxer de configuració de SSH:
      Bash
      1
      sudo nano /etc/ssh/sshd_config
    • Canvia la línia Port 22 a un número de port no usat (per exemple, Port 2222).

    Després, reinicia el servei SSH:

    Bash
    1
    sudo systemctl restart sshd

    2. Deshabilitar l'accés com a root

    L'accés SSH com a root és perillós. Deshabilita'l per obligar els usuaris a autenticar-se amb un compte normal i després usar sudo.

    • Al fitxer de configuració /etc/ssh/sshd_config, busca la línia:
      Bash
      1
      PermitRootLogin yes
    • Canvia-la a:
      Bash
      1
      PermitRootLogin no

    3. Usar autenticació per clau pública

    Deshabilita l'autenticació per contrasenya i usa claus SSH per autenticar-te.

    • Genera una clau SSH al teu ordinador client (si no en tens una):
      Bash
      1
      ssh-keygen -t rsa -b 4096
    • Després, copia la clau pública al servidor:
      Bash
      1
      ssh-copy-id usuari@server_ip
    • Finalment, deshabilita l'autenticació per contrasenya editant el fitxer /etc/ssh/sshd_config i canvia:
      Bash
      1
      PasswordAuthentication yes
      a:
      Bash
      1
      PasswordAuthentication no

    4. Usar SSH Protocol 2

    SSH té dues versions de protocol, i la versió 1 és considerada insegura. Assegura't que estàs utilitzant la versió 2:

    • Al fitxer /etc/ssh/sshd_config, assegura't que hi hagi aquesta línia:
      Bash
      1
      Protocol 2

    5. Restricció per IP (Firewall)

    Utilitza el tallafocs per permetre només connexions SSH des de determinades adreces IP.

    • Amb ufw (si està instal·lat):
      Bash
      1
2
      sudo ufw allow from IP_a_permetre to any port 2222
sudo ufw enable

    6. Temps d'inactivitat i desconnexió automàtica

    Configura un temps d'inactivitat després del qual la connexió SSH es tancarà automàticament.

    • Edita el fitxer /etc/ssh/sshd_config i afegeix aquestes línies:
      Bash
      1
2
      ClientAliveInterval 300
ClientAliveCountMax 2

    7. Deshabilitar X11 Forwarding

    Si no necessites l'accés remot gràfic, deshabilita la redirecció de X11.

    • Busca i canvia la línia següent a /etc/ssh/sshd_config:
      Bash
      1
      X11Forwarding yes
      a:
      Bash
      1
      X11Forwarding no

    8. Limitar els usuaris que poden connectar-se

    Si vols permetre només a certs usuaris que accedeixin via SSH, afegeix aquesta línia al fitxer de configuració:

    Bash
    1
    AllowUsers usuari1 usuari2

    9. Configuració de Fail2Ban

    Installar Fail2Ban pot ajudar a bloquejar les IP que intenten forçar l'accés al teu servidor SSH mitjançant múltiples intents fallits.

    Bash
    1
    sudo apt-get install fail2ban

    Aquestes pràctiques de hardening ajudaran a enfortir la seguretat del teu servei SSH i evitar atacs externs.

    Activitat pràctica

    Part 1

    Hardering

    Aplicar les configuracions que s'han explicat en la part anterior al teu servidor ssh.

    • Una de les mesures de seguretat recomanades és canviar el port d'escolta dels serveis per ports alternatius que no siguen superiors al 1024, els anomenats puertos well-known. Tria un port alternatiu; assegura't que no estiga en ús en el teu sistema i que no tingues previst utilitzar-lo, i configura el teu servidor perquè escolte les peticions per ell.

    • Configura openssh-server perquè no mantinga obertes les connexions que no han iniciat sessió durant més temps del necessari.

    • Creus que tots els usuaris del sistema necessiten tenir accés via SSH al servidor? Normalment no és així. Pensa en una estructura d'usuaris o grups i configura el servidor perquè permeta l'accés a l'usuari ubuntu i denegue l'accés a altres usuaris com els alumnes, que estaran dins d'un grup, però no als usuaris que estiguen fora del grup.

    Part 2

    Sense contrasenya

    Fins ara, sempre ens hem connectat al servidor SSH d'Ubuntu Server autenticant-nos amb usuari/contrasenya. L'autenticació mitjançant clau pública i clau privada dota el servidor SSH d'un plus de seguretat al nostre sistema, ja que, entre altres coses, és més resistent a atacs per força bruta o a robatoris de contrasenyes per descuit.

    Lliurament

    Puja un fitxer PDF amb les següents captures de pantalla:

    1. Canvis realitzats en l'arxiu de configuració del servidor SSH.

    2. Comanda que copia la clau pública de admin01 directament del client al servidor i la deixa preparada per al seu ús. No oblides que s'ha de veure el resultat.

    3. Comanda que copia la clau pública de l'usuari que has creat del client al servidor. No oblides que s'ha de veure el resultat.

    4. Comanda que mostra l'arxiu de claus autoritzades de l'usuari que has creat al servidor. Atenció, no es demana que mostre el contingut, sinó que mostre el nom de l'arxiu i els seus detalls.
