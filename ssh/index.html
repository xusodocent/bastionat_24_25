    SSH

    Tutorial complet sobre SSH

    SSH (Secure Shell) és un protocol criptogràfic que permet connectar-se de manera segura a una màquina remota per gestionar-la, executar comandes, transferir arxius, i molt més. És àmpliament utilitzat pels administradors de sistemes, desenvolupadors i qualsevol persona que treballi amb servidors remots.

    Què és SSH?

    SSH proporciona autenticació i encriptació per assegurar la comunicació entre dos sistemes. Es pot utilitzar en sistemes Linux, macOS i Windows (utilitzant clients SSH com PuTTY o la línia de comandes de PowerShell a Windows 10+).

    Components de l'ecosistema SSH:

    1. Client SSH: Un programa que s'utilitza per iniciar una connexió SSH des d'un dispositiu local a un dispositiu remot.
    2. Servidor SSH: El servidor al qual ens connectem de forma remota. Per exemple, un servidor Linux.
    3. Port: SSH utilitza per defecte el port 22, però es pot configurar per a altres ports.

    Passos per utilitzar SSH:

    1. Instal·lar SSH

    a) Linux i macOS

    SSH sol estar preinstal·lat en la majoria dels sistemes Linux i macOS. Pots verificar-ho executant:

    Bash
    1
    ssh -V

    Si no està instal·lat, ho pots fer amb:

    Debian/Ubuntu:

    Bash
    1
2
    sudo apt update
sudo apt install openssh-client

    CentOS/Fedora:

    Bash
    1
    sudo yum install openssh-clients

    b) Windows

    Windows 10 i posteriors tenen SSH preinstal·lat a PowerShell. Pots verificar-ho obrint PowerShell i executant:

    PowerShell
    1
    ssh

    Si no tens SSH instal·lat, pots instal·lar el client PuTTY o utilitzar Windows Subsystem for Linux (WSL) per executar comandes Linux directament des de Windows.

    2. Connectar-se a un servidor remot

    Per connectar-te a una màquina remota via SSH, necessites l'adreça IP o el nom de domini del servidor i les teves credencials (nom d'usuari i contrasenya o clau SSH).

    La comanda bàsica per establir una connexió és:

    Bash
    1
    ssh usuari@servidor

    Per exemple, per connectar-te a un servidor amb IP 192.168.1.100 amb l'usuari root:

    Bash
    1
    ssh root@192.168.1.100

    Si és la primera vegada que et connectes al servidor, t'apareixerà un avís per acceptar la clau del servidor:

    Text Only
    1
2
3
    The authenticity of host '192.168.1.100 (192.168.1.100)' can't be established.
ECDSA key fingerprint is SHA256:xxxx.
Are you sure you want to continue connecting (yes/no)?

    Escriu yes per continuar i, si utilitzes autenticació amb contrasenya, t'apareixerà un camp per introduir la contrasenya de l'usuari.

    3. Autenticació amb clau SSH

    L'ús de claus SSH és més segur que utilitzar contrasenyes. Les claus SSH són parelles de claus pública i privada que identifiquen un usuari sense necessitat d'introduir una contrasenya.

    a) Generar claus SSH

    Per generar una parella de claus (pública i privada) al teu sistema local, utilitza la comanda:

    Bash
    1
    ssh-keygen -t rsa -b 4096

    Això et demanarà on vols guardar la clau. Pots deixar-ho per defecte (~/.ssh/id_rsa) i, si vols, pots protegir la clau amb una contrasenya.

    b) Copiar la clau pública al servidor remot

    Una vegada generada la clau, pots copiar la clau pública al servidor remot amb la següent comanda:

    Bash
    1
    ssh-copy-id usuari@servidor

    Això permetrà al servidor reconèixer el teu sistema local i acceptar connexions sense que necessitis proporcionar una contrasenya.

    4. Opcions avançades de SSH

    a) Utilitzar un port diferent

    Si el servidor SSH està configurat per escoltar en un port diferent al 22, pots especificar el port amb l'opció -p:

    Bash
    1
    ssh -p 2222 usuari@servidor

    b) Reenviament de ports (Port Forwarding)

    SSH també permet reenviar ports per accedir a serveis privats a través de la connexió SSH segura.

    Reenviament de port local:

    Si vols accedir a un servidor local en un port remot (per exemple, un servidor web a localhost:8000 en el servidor remot), pots reenviar el port així:

    Bash
    1
    ssh -L 8080:localhost:8000 usuari@servidor

    Després d'això, pots accedir a localhost:8080 en el teu navegador local, i SSH reenviarà el tràfic al servidor remot en el port 8000.

    c) Execució remota de comandes

    Pots utilitzar SSH no només per connectar-te de forma interactiva, sinó també per executar comandes remotes sense iniciar una sessió completa:

    Bash
    1
    ssh usuari@servidor 'comanda_a_executar'

    Per exemple:

    Bash
    1
    ssh root@192.168.1.100 'uptime'

    5. Transferir arxius amb SCP

    SSH també es pot utilitzar per transferir arxius amb SCP (Secure Copy Protocol). La sintaxi per copiar fitxers entre màquines és:

    Bash
    1
    scp fitxer_local usuari@servidor:/ruta/desti

    Per exemple, per copiar un fitxer document.txt al servidor remot:

    Bash
    1
    scp document.txt root@192.168.1.100:/home/root

    Pots copiar directoris recursivament utilitzant l'opció -r:

    Bash
    1
    scp -r carpeta_local usuari@servidor:/ruta/desti

    6. Seguretat en SSH

    a) Desactivar el login amb contrasenya

    Per millorar la seguretat del teu servidor SSH, pots desactivar l'autenticació amb contrasenya i permetre només l'accés mitjançant claus SSH. Afegeix o modifica aquestes línies en el fitxer /etc/ssh/sshd_config del servidor:

    Bash
    1
    PasswordAuthentication no

    Després reinicia el servei SSH:

    Bash
    1
    sudo systemctl restart sshd

    b) Canviar el port per defecte

    Canviar el port per defecte (22) pot dificultar una mica els atacs per força bruta. Modifica el fitxer /etc/ssh/sshd_config i canvia la línia:

    Bash
    1
    Port 2222

    Després reinicia el servei SSH.

    7. Depuració de problemes en SSH

    Per depurar una connexió SSH fallida, pots utilitzar el mode detallat (verbose mode):

    Bash
    1
    ssh -v usuari@servidor

    Això mostrarà informació detallada sobre la connexió i ajudarà a identificar problemes.
