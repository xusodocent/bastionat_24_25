    PRÀCTICA IPTABLES

    Tallafocs - IPtables

    1. Escenari del Laboratori

    Maquinària Utilitzada:

    • Màquina real: 10.10.10.x
    • Firewall (FW): 192.168.122.150 - 172.16.1.128
    • Client: 172.16.1.129

    Configuració dels Sistemes Operatius:

    • Host extern: Màquina real.
    • Client: Ubuntu Desktop 20.04 o superior.
    • FW: Ubuntu Server 20.04 o superior (Apache i SSH).

    Subxarxes:

    • Internet: 10.10.10.0/24
    • Xarxa privada: 172.16.1.0/24

    Passos Inicials:

    1. Posa el tallafocs com a porta d'enllaç al client, és a dir, gateway: 172.16.1.128

    Si ho preferiexes configura les rutes per defecte al client des del terminal:

    Bash
    1
    ip route add default via 172.16.1.128

    1. Activa el reenviament de paquets al Firewall:
      Bash
      1
      echo 1 > /proc/sys/net/ipv4/ip_forward
    2. Instal·la els serveis necessaris al Firewall:
      Bash
      1
      sudo apt install openssh-server apache2

    2. Exercicis Inicials

    Exercici 1

    Configura la política d'eixida per defecte per bloquejar el tràfic:

    Bash
    1
2
    sudo iptables -P OUTPUT DROP
ping localhost

    Exercici 2

    Restaura la política inicial:

    Bash
    1
2
    sudo iptables -P OUTPUT ACCEPT
ping localhost

    Exercici 3

    Bloqueja el tràfic local cap al tallafocs:

    Bash
    1
2
    sudo iptables -A INPUT -i lo -j DROP
ping localhost
    Restaura el tràfic local:
    Bash
    1
    sudo iptables -D INPUT -i lo -j DROP

    Exercici 4

    Bloqueja l'accés HTTP des del tallafocs:

    Bash
    1
2
    sudo iptables -A OUTPUT -p tcp --dport 80 -j DROP
curl www.upv.es
    Restaura l'accés:
    Bash
    1
    sudo iptables -D OUTPUT -p tcp --dport 80 -j DROP

    Exercici 5

    Permet la comunicació per SSH al firewall:

    Bash
    1
    sudo iptables -A OUTPUT -p tcp --dport 22 -j ACCEPT
    Comprova el funcionament.

    Rebutja peticions de ping al Firewall:

    Bash
    1
    sudo iptables -A INPUT -p icmp -j REJECT
    Comprova el funcionament.

    Exercici 6

    Denega l'accés SSH:

    Bash
    1
    sudo iptables -A INPUT -p tcp --dport 22 -j DROP

    Exercici 7

    1. Comprova el ping al servidor:
      Bash
      1
      ping 192.168.122.x
    2. Bloqueja el ping al Firewall:
      Bash
      1
      sudo iptables -A INPUT -p icmp -j DROP

    3. Creació d'un Script de Tallafocs

    Esborrar regles anteriors:

    Bash
    1
2
    iptables -F
iptables -X

    Configurar les polítiques per defecte:

    Bash
    1
2
3
    iptables -P INPUT ACCEPT
iptables -P OUTPUT ACCEPT
iptables -P FORWARD ACCEPT

    Acceptar tràfic local:

    Bash
    1
2
    iptables -A INPUT -i lo -j ACCEPT
iptables -A OUTPUT -o lo -j ACCEPT

    Habilitar NAT:

    Bash
    1
2
    iptables -t nat -A POSTROUTING -s 172.16.1.0/24 -o eth0 -j MASQUERADE
echo 1 > /proc/sys/net/ipv4/ip_forward
    Introduïr totes les regles en un script i executar-lo

    Exercici 8

    • Comprovar que des del client tenim accés al servidor web del firewall i al servidor web de l'adreça IP del servidor de classe.
    • Denegar l'accés al servidor web del firewall.
    • Denegar l'accés només al servidor web de la classe.
    • Tindrem accés a un altre servidor web, com Google, UPV, etc.

    Exercici 9

    Exercici 9

    Des del client no es podrà fer SSH al firewall, però sí al servidor de classe.

    Exercici 10

    Utilitzant el mateix client (canviarem la IP), configurarem el tallafocs perquè el client 172.16.1.130 no tingui accés a Internet. Un vegada comprovat, tornarem a la IP original (172.16.1.129) i comprovarem que tenim connexió.

    Exercici 11

    Des de fora de la xarxa externa 192.168.88.0, només es podrà accedir al servidor web del firewall, però no tindrem accés a res de la xarxa interna. Ni es podrà fer ping ni accedir al servidor SSH.

    Exercici Extra

    Implementa una política de denegació general, però permet: - Navegar des del client 172.16.1.129. - Accedir al servidor SSH del Firewall. - La resta està prohibit
