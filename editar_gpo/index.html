    Editar GPOs

    Editar GPOs

    Edició

    Per editar les Group Policy Objects (GPOs) a un entorn d'Active Directory, cal seguir una sèrie de passos utilitzant les eines de Group Policy Management Console (GPMC) o les eines locals de política de grup. Aquí tens una guia per editar les GPOs de manera eficient:

    1. Accedir a la consola de Group Policy Management (GPMC)

    La Group Policy Management Console (GPMC) és l'eina centralitzada per gestionar les GPOs en Active Directory. Aquí pots crear, modificar i aplicar polítiques a les unitats organitzatives (OU) i dominis.

    Passos:

    1. Obrir GPMC:

    2. Obre el Server Manager a Windows Server.

    3. A la barra lateral esquerra, selecciona Tools (Eines) i després Group Policy Management (Gestió de polítiques de grup).

    4. Alternativament, pots obrir GPMC directament escrivint gpmc.msc a la finestra de Run (Windows + R).

    5. Localitzar l'objecte de política de grup (GPO):

    6. A la finestra de GPMC, navega pel domini o la unitat organitzativa (OU) on vols aplicar la política.

    7. Fes clic a l'OU o domini desitjat per veure les GPOs existents associades a aquesta ubicació.

    8. Crear o editar una GPO:

    9. Crear una nova GPO:
      Fes clic dret a la carpeta Group Policy Objects i selecciona New per crear una nova GPO. Assigna un nom clar i descriptiu.

    10. Editar una GPO existent:
      Fes clic dret sobre la GPO que vols modificar i selecciona Edit per obrir l'Editor de Política de Grup.

    2. Editar les GPOs:

    L'Editor de Política de Grup és l'eina per modificar les polítiques de seguretat, usuaris, contrasenyes, etc.

    Passos generals per editar GPOs:

    1. Obrir l'Editor de Política de Grup:

    Un cop seleccionada la GPO a GPMC, fes clic a Edit i s'obrirà l'Editor de polítiques de grup (Group Policy Management Editor).

    1. Configurar les polítiques de seguretat:

    Les GPOs que vols modificar es troben en les següents seccions dins de l'editor:

    • Polítiques de contrasenya i bloqueig de comptes:
      Camina per la ruta:
      Computer Configuration -> Policies -> Windows Settings -> Security Settings -> Account Policies -> Password Policy i Account Lockout Policy.
    • Polítiques d'auditoria:
      Camina per la ruta:
      Computer Configuration -> Policies -> Windows Settings -> Security Settings -> Advanced Audit Policy Configuration.
    • Polítiques d'accés i protecció dels dispositius:
      Camina per la ruta:
      Computer Configuration -> Policies -> Administrative Templates -> System -> Device Guard.
    • Polítiques de protecció d'administradors:
      Camina per la ruta:
      Computer Configuration -> Policies -> Windows Settings -> Security Settings -> Local Policies -> User Rights Assignment.

    • Polítiques de seguretat de la xarxa i registre:
      Camina per la ruta:
      Computer Configuration -> Policies -> Windows Settings -> Security Settings -> Network Security i Windows Settings -> Security Settings -> Registry.

    • Modificar les polítiques:

    Un cop localitzada la política que vols modificar (per exemple, la força de les contrasenyes), fes doble clic sobre ella i ajusta la configuració segons les teves necessitats. Algunes polítiques poden ser Enabled (activades), Disabled (desactivades) o Not Configured (no configurades).

    • Per exemple, per canviar la complexitat de la contrasenya, selecciona Enabled i aplica les condicions desitjades (com a mínim 12 caràcters, amb majúscules, minúscules, números i símbols).

    • Guardar els canvis:

    Un cop modificades les polítiques, tanca l'Editor de polítiques de grup. Els canvis es desaran automàticament.

    3. Aplicar la GPO a una unitat organitzativa o domini:

    Després de crear o editar una GPO, és necessari associar-la a una unitat organitzativa (OU) o al domini per aplicar-la als usuaris i equips corresponents.

    Passos:

    1. Associar la GPO a una unitat organitzativa o domini:
    2. A GPMC, selecciona la GPO que vols aplicar.
    3. Fes clic dret i selecciona Link an Existing GPO.

    4. A la finestra que apareix, selecciona l'OU o domini al qual vols aplicar la GPO.

    5. Force Apply (Aplicació forçada) (si és necessari): Si la GPO no es reflecteix immediatament, pots forçar l'actualització executant la comanda gpupdate /force des de la línia de comandes en el client o servidor objectiu.

    4. Revisar l'efecte de la GPO:

    Després d'aplicar les GPOs, pots revisar si les polítiques s'han aplicat correctament als usuaris i sistemes.

    Passos:

    1. Revisar els resultats amb la comanda gpresult:

    2. Executa gpresult /r des d'una finestra de comandes al client o servidor per veure les GPOs aplicades.

    3. Utilitzar el Group Policy Modeling** a GPMC per simular l'aplicació de GPOs a usuaris i sistemes específics abans d'aplicar-los de manera real.

    5. Eines alternatives:

    Si no tens accés al servidor per modificar les GPOs de manera centralitzada, pots utilitzar l'eina Local Group Policy Editor (gpedit.msc) en màquines locals, però aquesta només afecta la màquina on s'aplica, no al conjunt de dominis o OU com GPMC.

    Consells addicionals:

    • Documenta els canvis: Cada vegada que editis una GPO, és bo documentar els canvis per mantenir un historial de les polítiques aplicades.

    • Prova en entorns de desenvolupament: Abans de fer canvis en un domini de producció, és recomanable provar-los en un entorn de proves o en una màquina de test per evitar impactes no desitjats.

    • Revisa regularment les polítiques: Les polítiques de seguretat haurien de ser revisades i ajustades periòdicament per garantir la seva efectivitat.

    Seguint aquests passos, podràs configurar i modificar les GPOs per augmentar la seguretat en el teu domini Active Directory.
