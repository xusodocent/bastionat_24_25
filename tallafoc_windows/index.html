    Tallafoc Windows

    Pràctica d'ús del firewall de Windows

    El firewall de Windows és capaç de permetre o bloquejar les connexions entrants i sortints, tant les configurades per defecte com les que nosaltres configurem o modifiquem.

    Si volem evitar o permetre una connexió des de fora cap al nostre equip, haurem de registrar una regla d'entrada.

    Al contrari, si volem impedir una connexió des del nostre equip cap a fora, caldrà definir una regla de sortida.

    Vegem dos exemples en el Firewall de Windows per veure com funciona:

    • Evitar/Permetre que ens enviïn pings.
    • Permetre que només una IP específica es pugui connectar per RDP.

    Accés al Firewall de Windows

    Per accedir al firewall de Windows hem de cercar "Firewall de Windows Defender" i triar la primera opció.

    Veurem si tenim el firewall actiu i si estem connectats a xarxes de domini, privades o públiques.

    A les opcions de la part esquerra fem clic a "Configuració avançada" i ens apareixerà un menú amb un resum del firewall i dels perfils amb els seus permisos corresponents.

    Evitar / Permetre que ens enviïn pings

    Com hem comentat al principi, al firewall de Windows Defender vénen predefinides una sèrie de regles per defecte o predeterminades.

    Aquestes regles predefinides tenen a veure amb els serveis o les aplicacions que les utilitzen i normalment quan instal·lem una nova aplicació, si necessita realitzar algun tipus de connexió entrant o sortint, ens informarà de la seva configuració.

    La regla que gestiona l'ús del "ping" està relacionada amb les peticions ECO del protocol ICMP, i solen estar configurades i actives per defecte.

    Com en aquest cas el que volem és: evitar / permetre l'ús de "ping", estem admetent o rebutjant connexions externes cap al nostre equip, per tant, és una regla d'entrada.

    Així que el primer que farem és anar a la secció de Regles d'entrada.

    Ara hem de cercar la regla "Arxius i impressores compartits (petició eco ICMPv4 d'entrada)".

    Potser ens apareguin una o dues regles, una habilitada i l'altra deshabilitada, depenent de si estem connectats a un domini o no. Si no té el "tic verd" significa que està deshabilitada. Per habilitar-la cal fer clic dret sobre la regla i seleccionar "Habilitar regla".

    Ara si enviem ping des d'un altre equip a aquest, ens retornarà una resposta.

    D'altra banda, si la volem deshabilitar, fem clic dret sobre la mateixa regla que acabem d'habilitar i seleccionem "Deshabilitar regla". Ara si fem ping no rebreu resposta.

    Permetre que una IP específica es pugui connectar per escriptori remot (RDP)

    El protocol RDP utilitza per defecte el port 3389/tcp, la seva utilització està molt estesa per a les connexions remotes a través d'escriptori remot.

    Tanmateix, si no està ben configurat, podem ser víctimes d'atacs. Per això cal especificar quines adreces IP volem que es connectin.

    En aquest cas, el Firewall de Windows també té unes regles predefinides sobre l'escriptori remot. Hem de cercar "Escriptori remot – Mode usuari (TCP d'entrada)" i també el de UDP.

    Fem clic dret sobre el de TCP i li donem a "Propietats". S'obrirà un menú i pitjarem a "Àmbit".

    A la part inferior, a "Adreça IP remota" hem d'afegir les IP que volem que es connectin. Per això li pitjarem a "Afegir". Podem afegir una IP o un rang d'IP.

    Després d'afegir la IP li donarem a "Aplicar" i a "Acceptar". Tot això ho farem igualment amb el protocol UDP.
