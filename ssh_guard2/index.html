    SSH-Guard

    SSH-Guard

    SSH-Guard és una eina per a la protecció contra atacs de força bruta a serveis com SSH, ajudant a detectar i bloquejar IPs sospitoses després d'un cert nombre d'intents fallits de connexió.

    Activitat pràctica: Configuració i ús d'SSHGuard per protegir el servei SSH

    Objectiu:

    Configurar SSHGuard per detectar intents d'atac de força bruta i bloquejar automàticament les adreces IP sospitoses que intentin accedir repetidament sense èxit al servei SSH.

    Requisits previs:

    • Un servidor Linux amb accés a SSH (qualsevol distribució que suporti SSHGuard, com Ubuntu, Debian, CentOS, etc.).
    • Accés com a superusuari (root) o permisos de sudo per fer canvis de configuració.

    Passos de l'activitat:

    1. Instal·lació d'SSHGuard

    Per començar, instal·la SSHGuard al teu servidor. La comanda dependrà de la distribució Linux que estiguis utilitzant:

    • Ubuntu/Debian:

      Bash
      1
2
      sudo apt-get update
sudo apt-get install sshguard

    • CentOS/RHEL:

      Bash
      1
      sudo yum install sshguard

    • Fedora:

      Bash
      1
      sudo dnf install sshguard

    2. Configuració bàsica d'SSHGuard

    Després de la instal·lació, configura SSHGuard perquè monitoritzi el fitxer de registre d'errors de connexió SSH (/var/log/auth.log o equivalent segons la teva distribució).

    • Ubuntu/Debian: Generalment, el registre d'autenticació es troba a /var/log/auth.log.

    No cal modificar fitxers de configuració d'SSHGuard per defecte, ja que l'eina sap on trobar els registres per defecte. No obstant això, si vols personalitzar la configuració, pots editar el fitxer de configuració:

    Bash
    1
    sudo nano /etc/sshguard/sshguard.conf

    Dins aquest fitxer, podràs modificar aspectes com el nombre d'intents fallits permesos abans de bloquejar una IP, o la durada del bloqueig.

    3. Integració amb el sistema de tallafocs (optional)

    SSHGuard funciona amb diversos tallafocs com iptables, nftables, o pf. En moltes distribucions, SSHGuard es configura automàticament per treballar amb iptables o nftables.

    Per verificar-ho:

    • Iptables:
      Bash
      1
      sudo iptables -L

    Hauries de veure una regla d'SSHGuard a la llista, que indica que l'eina bloquejarà adreces IP malicioses.

    4. Provar la configuració de SSHGuard

    Ara que tens SSHGuard instal·lat i configurat, fes una prova pràctica per veure com reacciona davant d'intents fallits de connexió:

    • Obre una terminal i intenta connectar-te al servidor amb un nom d'usuari o contrasenya incorrecta repetidament (uns 4-5 intents).
    Bash
    1
    ssh usuari_incorrecte@ip_del_servidor

    Després de diversos intents fallits, SSHGuard hauria de bloquejar la teva IP temporalment.

    5. Verificar l'activitat de SSHGuard

    Després de fer els intents fallits, pots verificar si SSHGuard ha bloquejat la teva IP consultant els registres:

    Bash
    1
    sudo tail -f /var/log/syslog

    Hauries de veure alguna cosa com això:

    Text Only
    1
    sshguard[xxxx]: Blocking 192.168.x.x after 4 attempts.

    Això vol dir que SSHGuard ha detectat els intents d'atac i ha bloquejat l'adreça IP sospitosa.

    6. Desbloquejar manualment una IP (opcional)

    Si vols desbloquejar manualment una IP bloquejada per SSHGuard, pots fer-ho amb les següents comandes segons el tallafocs que facis servir:

    • Iptables:
      Bash
      1
      sudo iptables -D sshguard -s 192.168.x.x -j DROP

    7. Configuració avançada (opcional)

    Si vols canviar la durada del bloqueig o ajustar els paràmetres de detecció, pots modificar-ho al fitxer de configuració /etc/sshguard/sshguard.conf:

    • Exemple: canviar la durada del bloqueig a 1 hora (3600 segons):
      Bash
      1
      BLOCK_TIME=3600

    Després de modificar la configuració, reinicia SSHGuard per aplicar els canvis:

    Bash
    1
    sudo systemctl restart sshguard
